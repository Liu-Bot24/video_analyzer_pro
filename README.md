Video Analyzer Pro 是一款基于视觉大语言模型 (VLM) 的高性能视频审计与全场景分类工具。它专为大规模视频库设计，通过高度自定义的提示词 (Prompt) 逻辑，将海量视频的“内容理解、合规审查、自动分类与重命名”一站式自动化。

## 🌟 为什么选择它？

- **工业级审计精度**：默认内置“视觉内容审计员”指令，能精准识别现实中的肢体冲突、交通纠纷、暴露内容及各类不适画面，而非简单的关键词过滤。
- **全场景无限扩展**：**核心逻辑完全由 Prompt 驱动**。您可以随时通过配置文件，将它变身为“猫咪行为分析师”、“交通违章审计员”或“短视频题材分类器”。
- **时长自适应截帧**：首创动态阶梯算法。无论是 5 秒短视频还是 1 小时电影，程序都会自动匹配最佳截帧数，确保不遗漏关键动作，同时为您节省 Token。
- **极致的鲁棒性**：配置热切换（无需重启）、CSV 防占用自动重试、多线程安全锁、环境自愈脚本（自动装 Python/FFmpeg，仅适用于Windows）。

## 📂 应用场景示例

1.  **安全合规审计（核心案例）**：自动化筛选视频库，精准定位并分拣出暴力殴打、色情擦边或令人不安的内容。
2.  **交通事故分析**：通过精细化的 Prompt，从海量行车记录仪画面中提取冲突、拉扯、碰撞等特定事件。
3.  **素材库智能分拣**：自定义“自然风景、人物专访、工业生产”等规则，让 AI 自动将杂乱的素材库按题材整理进对应文件夹。
4.  **数字化标签管理**：为每个视频自动生成 15 字精准标题，并汇总至 CSV 表格，建立可搜索的数字化视频档案。

## 🚀 快速开始

### 1. 准备环境与 API 建议
- 确保系统连接互联网。
- 准备一个支持 Vision 能力的 OpenAI 兼容接口（例如[SiliconFlow 平台](https://cloud.siliconflow.cn/i/My0p5Jgs)）。
- **性能提示**：建议根据预算和速度需求在 `config.yaml` 中选择不同的模型梯度，如 `THUDM/GLM-4.1V-9B-Thinking`（支持免费使用）、`Qwen/Qwen3-VL-32B-Instruct`（TPM较高）、`GLM-4.6V`、`Qwen3-VL-30B-A3B` 等。但需要注意，由于不同计费模式的模型具有不同的 **TPM (Tokens Per Minute)** 频率限制，如追求处理速度还是建议自己购买适配的 AI 模型服务。
- 如对模型表现稳定性要求宽松，可使用[此版本](https://github.com/Liu-Bot24/video_analyzer_pro/tree/multi-model-threads)。支持配置多个薅羊毛得来的免费/低价模型并发线程进行异步处理，突破TPM限制，提高速度。

### 2. 配置与运行
1.  **编辑 config.yaml**：填入您的 API 密钥和视频目录。
2.  **自定义分类 (可选)**：在 categories 下修改或添加您想识别的任何内容（如：宠物种类、动作类型等）。
3.  **启动程序**：
    - **Windows 环境**：双击 **run_analyzer.bat**，它会自动检查并部署 Python/FFmpeg 环境后启动任务。
    - **Mac / Linux 环境**：本项目核心逻辑完全跨平台。请先执行 pip install -r requirements.txt，然后终端运行 python main.py 即可。

## ⚙️ 配置文件核心项说明 (`config.yaml`)

本项目所有的核心逻辑与参数均由 `config.yaml` 驱动，支持极致的自定义配置：

### [API 核心连接配置]
| 配置项 | 说明 |
| :--- | :--- |
| `api.key` | API 访问密钥（支持各大兼容 OpenAI 格式的平台）。 |
| `api.base_url` | API 请求的基础 URL 地址。 |
| `api.model` | 调用的视觉大模型名称，支持随时热切换。 |
| `api.timeout` | 单次网络请求的超时上限（秒），防大图传输卡死，建议 180s。 |
| `api.max_retries` | 接口请求失败（如网络波动/限流）后的自动重试次数。 |
| `api.temperature` | 采样温度 (0.0-1.0)，建议调低 (如 0.1) 保持分类输出的严谨性。 |
| `api.max_tokens` | 允许模型生成的最大 Token 长度。 |

### [视频预处理配置]
| 配置项 | 说明 |
| :--- | :--- |
| `video.source_dir` | 待处理的原始视频存放目录（支持直接粘贴 Windows 绝对路径）。 |
| `video.dynamic_frames` | **全自定义动态截帧阶梯**。格式如 `[时长上限(秒), 提取帧数]`，支持无限增加阶梯，程序会自动计算视频时长并匹配最佳帧数。 |
| `video.max_dimension` | 提取图片后的长边缩放上限，超过此像素值会自动等比缩小。 |
| `video.extensions` | 程序扫描的视频文件后缀名白名单（如 `.mp4`, `.avi` 等）。 |
| `video.auto_rename` | 开关：是否根据 AI 生成的“短标题”自动进行物理重命名。 |
| `video.keep_original_name` | **重命名策略**：设为 `true` 则生成 `短标题_原文件名.mp4`；设为 `false` 则仅保留 `短标题.mp4`（程序内置了完善的重名防覆盖后缀自增逻辑）。 |

### [分类审计定义与存储规则]
| 配置项 | 说明 |
| :--- | :--- |
| `categories` | **完全自定义的审计规则库**。包含任意个您定义的分类 Key（如 `violence`、`sex` 等），该 Key 将作为系统识别标签发给 AI。 |
| `path` | 识别结论为该类时，视频物理移动的目标子目录。 |
| `desc` | **发给 AI 的视觉判定准则**，通过详尽客观的特征描述引导 AI 做出精准分类。 |

### [系统运行行为配置]
| 配置项 | 说明 |
| :--- | :--- |
| `system.concurrency` | 并发处理的线程数量，建议根据您的 API 频率限制（RPM）进行调节，通常 1-4 较为安全。 |
| `system.csv_file` | 最终识别结果汇总电子表格的存放名称。 |
| `system.log_file` | 运行过程中的详细操作日志文件名。 |
| `system.prompt_template` | **核心大脑**：发送给 AI 的提示词模板，自动提取上方的 `desc` 规则完成动态注入。 |

## 🛠️ 技术保障

- **断点续传**：支持中断后重启，自动跳过已处理的文件。
- **碰撞防护**：重命名时若遇冲突，自动应用自增序号，绝不覆盖文件。
- **隐私保护**：仅发送视觉描述信息，自动剥离本地物理路径。
